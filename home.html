AB Testing

Version A & Version B

Null and Alternate Hypothesis

Metric 1: Time to Completion

Definition:the time it takes to complete the desired task (in this case, order cacti)
Null Hypothesis: Version A will have shorter time to completion than version B.
Alternative Hypothesis: Version A will not have shorter time to completion than version B.
Reasoning: A high time to completion for a certain task in a website may be an indication of difficulty
in navigating the website. A good website design should be able to have users to have a good idea of all of its
components at a first glance. (It shouldn't take the user a long time to go through all the elements of the website)
In my new website, I decided to stack everything in the middle instead of having them pushed to the left.
I thought having all the contents centered will make it easier for the users to have a quick glance at the website
and figure out where things are locate eaasier. If my thought was correct, the p value will be small enough (in this case,
less than 0.05) to reject the null hypothesis and show that it takes longer for the users to complete the task for version A
website than it would for version B website.

Metric 2: Return Rate

Definition: the number of times a user returned to the page
Null Hypothesis: Version A and Version B will not have significant difference in their return rate
Alternative Hypothesis: Version A and Version B will have significant difference in their return rate
Reasoning: A website having a high return rate may be an indication user being unsure of the cart content
(ex: how many things are in the cart, How many times the user clicked on certain item,...etc) I thought aligning
the elements to come to the center of the page would make it easier for the viewers to keep an eye on the cart and
the item log at the same time, thus reducing the return rate. If version B of the website turns out to be more
effective in reducing the return rate, the p value will turn out to be small enough to reject the null hypothesis

Data Collection

  • CS1300 students were given a list of heroku links to different websites that generates either version A or version B with 50/50 chance
  • Everyone were asked to add items until the cart goes over 150$ (threshold value) --> each student's log was recorded
  • Then, I manually calculated the time of entry and exit for each user and recorded them for further calculation. Return rate of each user was also recorded

Time to Completion

Conclusion: Since the P-value is greater than a (0.4671477551 > 0.05), we fail to reject the null hypothese.

Return Rate
Chi^2 value: 0.385714285714
P-value : 0.534560802035
Conclusion: Since the P-value is greater than a (0.534560802035 > 0.05), we fail to reject the null hypothese.
(In other words, we accept the Null Hypothesis) There was no significant difference in the return rate between the two versions of the website

Broad Takeaway

  • The result has limitations because of the small sample size. I could get ~25 users to test my website which is not enough to be the representation of the entire pool of users
  • From the result, I figured that the users had shorter time to completion when the elements were left-sided than when they were centered (contrary to my initial thought)
  • The results might have also been less accurate because the user experience that we collected during section didn't quite align with the actual user experience. (We were simply told to fill the cart over 150$ and move onto the next website. An acutal user of the website will take longer time navigating the website and moving on to the checkout page.
